Phase 4: Stress Testing & Production Hardening
--------------------------------------------------

[X] Fixed async DuckDuckGo search — rewrote search_async() to use asyncio.to_thread()
     - Problem: ddgs v9.x + duckduckgo-search v8.x both lack atext() async method
     - Fix: wraps sync DDGS.text() in asyncio.to_thread() — works with any library version
     - Import order: ddgs first (v9.x), fallback duckduckgo_search (v8.x)

[X] Fixed trafilatura output_format — "text" → "txt"
     - Problem: trafilatura v2.0.0 removed "text" format; raises AttributeError
     - Fix: output_format="txt" — silently killed all scraping (exception → None → empty)
     - All async scraping was returning 0 pages before this fix

[X] Fixed scraped content dedup merge — separate seen_scraped_urls set
     - Problem: scraped URLs were checked against seen_urls (already populated by results)
     - Fix: new seen_scraped_urls set — all scraped content was lost in merge
     - Now 40+ pages survive dedup and feed into Stage 4 reasoning

[X] Added reference numbering system — ref_id from Stage 3 merge
     - Each deduplicated article gets a sequential ref_id (1, 2, 3...)
     - Separate references list maps ref_id → {title, url} for the source table
     - Passed to reasoning_engine for AI citation prompt

[X] Added citation tracking — [N] format in AI output → cited source flag
     - REASONING_SYSTEM_PROMPT: instructs model to cite sources as [1], [2][3]
     - _extract_cited_refs(): regex extracts all [N] from AI JSON response
     - _build_reference_list(): marks each ref as cited=True/False
     - Local synthesis: marks all refs as cited (extractive = uses all sources)

[X] Added print_references_table() — Rich table of cited sources
     - Header: # | Source | URL
     - Color-coded: green ref_id if cited, overlay0 if not
     - Shows up to 10 cited sources + remaining count
     - Uses pink header, simple box style

[X] Resilience verification — all fallback paths confirmed:
     - Planner (Stage 0): no API key → returns [topic] → single sub-query
     - Planner (Stage 0): API fails → catches Exception → returns [topic]
     - Reasoning (Stage 4): no API key → _local_synthesis() via TextAnalyzer
     - Reasoning (Stage 4): AI fails → catches Exception → _local_synthesis()
     - Empty articles → graceful "No sources available" summary
     - Empty results → empty dicts/lists, no crashes

[X] End-to-end stress test results (topic: "how to build confidence"):
     - Stage 0 (Planning): 5 sub-queries in 4.0s via inclusionai/ring-2.6-1t:free
     - Stage 1 (Search): 50 sources across 5 queries in 53.8s
     - Stage 2 (Scrape): 40 pages scraped in 92.9s (aiohttp, semaphore 20)
     - Stage 3 (Merge): 49 deduplicated refs (URL + title dedup)
     - Stage 4 (Reasoning): 17.4s AI synthesis, confidence=high, 6 findings
     - Citations: 13/49 sources cited by reference number
     - Total time: ~2.5 min | No errors, smooth transitions

Details:
--------
- duckduckgo_client.py: asyncio.to_thread() keeps async pipeline pattern without atext()
- web_scraper.py: output_format="txt" aligns with trafilatura v2.0.0 API
- async_engine.py: seen_scraped_urls separate from seen_urls for proper merge
- async_engine.py: references list built alongside articles in Stage 3
- reasoning_engine.py: articles[:20] × 1200 chars + scraped[:10] × 1500 chars = ~25k tokens
- reasoning_engine.py: _extract_cited_refs uses re.finditer(r"\[(\d+)\]") on json.dumps(parsed)
- output_formatter.py: print_references_table() — 10 max + "+N remaining" overflow
- Test confirms all 3 languages supported: Arabic, French, English
